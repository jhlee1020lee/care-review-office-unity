# AI 활용 기록

작성일: 2026-06-05

## 개요

`돌봄지원 심사소` Unity 프로젝트 개발과 백업 과정에서 ChatGPT/Codex를 보조 개발 도구로 활용했다. 사용자는 구현 방향, 저장소 생성, 백업 진행 여부를 결정했고, AI는 로컬 프로젝트 점검, 코드 수정, 자산 생성, 테스트 실행, 문서화, Git 백업 작업을 수행했다.

## 대화 및 작업 요약

- 사용자는 게임에 건조한 사무실 톤의 배경음악과 효과음을 추가하고 싶다고 요청했다.
- AI는 외부 저작권 음원을 사용하지 않고, 로컬 합성 스크립트로 WAV 오디오 자산을 생성하는 방향을 제안하고 구현했다.
- AI는 Unity 프로젝트의 기존 `CareReviewGame.cs` 구조를 확인한 뒤, WAV 자산을 우선 로드하고 없을 때만 절차적 오디오를 폴백으로 쓰도록 수정했다.
- AI는 BGM 5종, SFX 20종을 생성하고 화면별 BGM 전환 및 UI/도장/사건/저장/오류 효과음을 연결했다.
- AI는 오디오 스모크 테스트를 확장해 자산 로드 여부, 샘플 수, 화면별 BGM 키, 음량 0 처리 등을 검증했다.
- 사용자는 작업 내용을 백업하기 위해 GitHub 저장소를 만들었고, AI는 로컬 git 초기화, `.gitignore` 작성, 커밋, 원격 저장소 푸시를 수행했다.
- 사용자의 요청에 따라 AI는 프로젝트 README를 작성하고 GitHub에 커밋 및 푸시했다.

## AI 활용 방식

- 코드 분석: 기존 Unity 스크립트, 문서, 프로젝트 구조를 읽고 수정 범위를 파악했다.
- 구현 보조: 오디오 로딩 로직, 재생 연결, 테스트 로직을 코드에 반영했다.
- 자산 생성: 저작권 음원 대신 Python 기반 합성 스크립트로 WAV 파일을 직접 생성했다.
- 품질 확인: Unity 빌드와 런타임 스모크 테스트를 실행해 변경사항이 동작하는지 확인했다.
- 문서화: README, 개발 로그, 릴리즈 체크리스트, Steamworks 준비 문서에 변경 내용을 반영했다.
- 버전 관리: 백업용 zip 생성, git 커밋, GitHub 원격 저장소 푸시를 도왔다.

## 검증 내용

- 생성된 WAV 자산은 44.1kHz, 16-bit PCM 형식으로 확인했다.
- 오디오 스모크 테스트에서 BGM 5개와 SFX 20개가 모두 로드되는 것을 확인했다.
- 기본 실행, 저해상도 UI, 리포트 도달 스모크 테스트를 통해 주요 화면 흐름이 유지되는지 확인했다.
- GitHub 저장소에는 소스, 문서, 설정 파일을 올리고, 빌드 결과물과 Unity 캐시 등 로컬 산출물은 제외했다.

## 한계

- AI는 자동 테스트와 로그 확인을 수행했지만, 실제 스피커로 5분 이상 플레이하며 듣는 수동 청음 검수는 별도로 완료하지 않았다.
- 최종 판단과 배포 여부는 사용자가 확인해야 한다.
