# 한국어 표시 문구 정리 내역

작성일: 2026-06-07
대상: 사용자에게 보이는 한국어 표시 문구 (UI 텍스트)
주 수정 파일: `Assets/Scripts/CareReviewGame.cs`

## 작업 원칙

- **로직 변경 금지**: 표시 문구와 이를 검증하는 자체 점검(`.Contains(...)`) 리터럴을 항상 **한 쌍으로 함께 교체**하여 동작은 그대로 유지.
- **식별자 보존**: 변수명·함수명·클래스명·저장 키·파일명은 일절 변경하지 않음. (`triage`/`Handoff`/`fallback` 등은 식별자에 다수 잔존하며 전부 보존)
- **레이아웃 보존**: 위치·크기 값 변경 없음. 라벨 길이도 동등 수준 유지.
- **의미 보존**: 뜻이 바뀌는 의역 없이 표기만 자연스러운 행정 용어로 정리.
- **톤**: 건조한 행정 사무실 / 공공기관 심사 업무 / 차분한 리포트.

## 3. 의도적으로 유지한 것 (올바른 현지화 판단)

번역하면 오히려 부자연스럽거나 의미가 깨지는 항목은 그대로 두었습니다.

- **표준 약어**: `UI`, `CSV`, `JSON`, `HTML`, `Markdown`, `CPU`, `GPU`, `RAM`, `FPS`, `ID`, `ms`, `AI`, `SHA256`, `zip`
- **고유명사**: `Steam`
- **파일 경로 / 키**: `Data/cases_day*.json`, 성과 키(`CARE_ANALYTICS_EXPORT` 등), CSV 컬럼명(`decision_rationale_id`)
- **하드웨어 키 이름**: `A` `B` `X` `Y` `LB` `RB` `View` `Menu` 등 컨트롤러/키보드 라벨
- **데이터 상태 키**: `stable` / `strained` / `high_pressure` (튜닝 리포트의 카테고리 식별자)

## 4. 점검 범위

- 화면 제목 14종, 메뉴·설정 버튼, 튜토리얼·첫 사례 가이드
- 시작 전 고지, 크레딧, 사례 데이터 JSON, 엔딩 내러티브
- 판단 피드백, 심사 기록, 후속 연락함, 기준표/기준 비교
- 리포트·운영 등급·결과 해석, 성과·캠페인 기록
- 지원 번들·환경 진단 산출물
- 한국어에 인접한 모든 영문 토큰을 정규식으로 전수 스캔
- 오탈자(되여/읍니다/할수 등) 검사 — 발견 없음

## 5. 확신이 없어 확인이 필요한 항목

1. **`포커스`(focus)** — `"장기 추천 포커스"`, `"캠페인 기록 포커스"` 등에 일관되게 쓰이지만 자체 점검 다수와 결합되어 이번엔 유지. 더 행정체로 바꾸려면 `선택`/`지정`/`초점`이 후보.
2. **`triage` → `분류`** — 의미상 `선별`도 가능. 선호 시 일괄 변경 가능.
3. **`압박도 → 압박` 방향** — 다수에 맞췄으나, 척도 느낌의 `압박도`를 선호하면 반대 방향도 가능.
4. **`OK` 상태 라벨** — 준비 표시(`로그 OK`/`설문 OK`/`환경 OK`/`경로 OK`)는 짧고 흔해 유지. 더 건조한 톤을 원하면 `충족`/`완료`로 변경 가능.
