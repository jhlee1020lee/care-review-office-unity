## 초보자 관점 검토

처음 켰을 때 기존 제출 스크린샷의 메인 메뉴는 분위기와 완성도는 좋지만, 초보자에게 "지금 뭘 해야 하는지"를 가장 먼저 말해주지는 않는다. 버튼은 `새 캠페인 시작`으로 충분히 이해 가능하지만, 좌측 설명은 로그/관리 설명이 중심이라 첫 행동 안내가 약하다.

이를 보완하기 위해 현재 소스에는 다음 개선이 들어가 있다.

- 새 저장 슬롯/경력 기록이 없는 상태에서는 메인 버튼을 `첫 근무 시작`으로 표시한다.
- 메인 메뉴 상태 문구가 `처음이면 [첫 근무 시작]을 누르세요. 4장 안내 후 첫 신청서를 함께 처리합니다.`로 첫 행동을 직접 안내한다.
- 완전 신규 상태의 메인 메뉴 상태 영역은 장기 목표/성과 문구 대신 `오늘 순서: 안내 확인 -> 문서 읽기 -> 기준 확인 -> 도장 선택`을 보여준다.
- 저장 기록이 있는 일반 메뉴 상태에서도 첫 줄에 `처음이면 N 새 캠페인 · 저장 있음 C 이어하기 · D 기준`을 표시해 교수/테스터 PC에 과거 저장이 있어도 시작 행동을 먼저 볼 수 있게 했다.
- 첫 실행 고지 마지막 문장이 `확인 후 메인 메뉴에서 첫 근무 시작을 누르면 4장 안내가 열립니다.`로 이어진다.
- 캠페인 시작 안내는 4장 구성으로 줄여 `오늘 할 일`, `읽는 순서`, `판단 기준`, `첫 사례 시작` 흐름을 만든다.
- 첫 사례 화면의 세부 가이드는 기존처럼 5단계로 유지해 실제 심사 화면의 정보량을 단계적으로 설명한다.
- 캡처 검토 후 첫 사례 5단계 설명문은 720p에서도 덜 빽빽하게 보이도록 더 짧은 행동형 문장으로 줄였다.
